Introduction

This week focused on mastering documentation practices using GitHub, establishing the foundation for our digital portfolio. As the Chinese proverb says, "The palest ink is better than the best memory," emphasizing the importance of documenting our work throughout the program.

We learned that documentation is a superpower that serves multiple purposes: helping us remember our work when returning to projects months later, building our professional portfolio, enabling collaboration with others, and serving as the primary method for submitting and getting graded on our work.

Documentation serves multiple important purposes

The week involved setting up essential tools including Git for version control, VS Code as our text editor, Git Bash (for Windows users) as our terminal, GitHub.com as our cloud platform, and SSH keys for secure authentication.

SSH Key Setup

We established secure password-less authentication with GitHub using SSH keys:

  1. Generated SSH key pair using: ssh-keygen -t ed25519 -C "cyriannayk@gmail.com"

Generating SSH keys in Git Bash

  1. Accepted default location and optionally set a passphrase
  2. Printed and copied the public key using: cat ~/.ssh/id_ed25519.pub
  3. Added the public key to GitHub in Settings → SSH and GPG keys

Adding SSH key to GitHub settings

Forking and Cloning

We practiced forking an existing repository to create our own copy, then renamed it using the format: your_name-unipodName.

Fork button on GitHub repository

We then cloned the repository to our local machine using Git Bash:

git clone git@github.com:your-Nyanbol Miyen/rNyanbol-Miyen-UniPod-South-Sudan.git

Cloning repository to local machine

Making and Pushing Changes

We learned the essential Git workflow:

  1. Opened our repository folder in VS Code
  2. Edited the README.md file, adding: ## Welcome to My UniPod Journey!

Editing README.md file in VS Code

  1. Used the three-step Git workflow:
    • git add README.md - Staged our changes
    • git commit -m "Update README with a welcome message" - Saved changes locally
    • git push origin main - Pushed changes to GitHub

GitHub Pages Deployment

We turned our repository into a live website using GitHub Pages:

  1. Went to repository Settings → Pages
  2. Selected "Deploy from a branch"
  3. Chose the main branch and root folder
  4. Saved settings
  5. Our site became live at: https://your-username.github.io/unipod-portfolio/

Reflection

This week was foundational in establishing our digital workflow and documentation practices. Learning Git and GitHub is crucial not just for this program but for any modern technical career. The concept of version control was initially challenging to grasp, but the three-step workflow (add, commit, push) became intuitive with practice.

Setting up SSH keys was particularly valuable as it provides a secure way to authenticate with GitHub without constantly entering passwords. This will streamline our workflow throughout the program.

Creating my first repository and deploying it with GitHub Pages was empowering. Seeing my work live on the web after following the steps made the abstract concepts concrete. The ability to fork repositories is especially powerful for collaboration and contributing to open-source projects.

Documentation, as emphasized this week, is indeed a superpower. It not only helps others understand our work but also serves as a personal reference when returning to projects. I'm beginning to see how this practice will be essential throughout the UniPod program and beyond.

This week's activities have established a solid foundation for my digital portfolio and documentation habits. I'm looking forward to applying these skills to future projects and continuing to build my GitHub profile as a professional portfolio.
